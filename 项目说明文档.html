智能空调控制面板 — 项目说明文档

480×480px 智能家居空调控制面板系统，含前端交互原型和后端 API 预留

本项目为智能家居空调控制面板的原型系统，面板尺寸 480×480px，适用于嵌入式触控屏或智能家居中控设备。系统采用前后端分离架构，前端使用 React 18 + TypeScript + Vite 6，后端使用 Express 4 + TypeScript，当前阶段前端使用 Mock 数据独立运行，后端 API 已预留完整接口供后续对接。通过 Vercel 自动部署（GitHub push 触发 re-deploy）。

四、前端核心组件

TemperatureDial（温度旋钮）

• 实现：SVG 半圆弧 + Pointer Events

• 交互：触摸/鼠标拖动，atan2 计算角度映射温度

• 防冲突：setPointerCapture + stopPropagation

• 范围：16–30°C，整数步进

• 视觉：渐变进度弧 + 刻度线 + 旋钮 + 中央大号数字

FanSpeedControl（风量调节，通风模式专用）

• 5 个风量档：1~5 档，垂直细长条（14×40）+ 档位数字

• 运行模式切换：自动 / 手动 两个 chip

• 手动模式：用户点击档位直接设置风量

• 自动模式：风扇持续转动 + 5 段"充电"跑马灯（从 0 累加到 5，2.5s 循环），整体不可交互

• 实现：纯 CSS 关键帧 + :nth-child() 设置不同 animation-delay（避开 transform: scaleY 黑屏风险）

视图架构

采用 横向 track + 覆盖层 混合架构：

横向 track（左/右滑动）：首页 → 总控面板 → 房间列表

覆盖层（直接替换 + 150ms opacity 渐变）：房间详情 / 全局设置

  • 3 个横向视图同时渲染在 app__track（300% 宽）容器中，通过 translateX 控制位置
  • 每个视图占 33.3334% 宽度
  • 房间详情 + 全局设置作为独立覆盖层（.app__overlay）：position: absolute; inset: 0，opacity: 0 → 1 渐变 150ms（无位移）
  • 覆盖层打开时 onSwipeLeft / onSwipeRight 传 undefined，自动禁用横向滑动
  • 房间详情页和设置页不显示顶部跑马灯（由覆盖层自然遮挡）

主题系统

三套主题通过 CSS 变量 + data-theme 属性切换：

[data-theme="tech-blue-purple"] { --bg-primary: #07091a; --accent: #6c5ce7; }
[data-theme="black-gold"]       { --bg-primary: #0a0a0a; --accent: #D4AF37; }
[data-theme="gray-white"]       { --bg-primary: #fafafa; --accent: #1a1a1a; }

切换时 0.35s 平滑过渡，所有颜色使用 CSS 变量，无硬编码色值。

亮度系统

• useBrightness hook 每分钟检测当前时间

• 白天：8:00-18:00 / 晚上：18:00-8:00

• 通过 #root { filter: brightness(X%) } 应用亮度

• 最低可读性保护：白天不低于 50%、夜间不低于 30%

• 跨时段预览：白天调夜间/夜间调白天滑块时，临时预览 3 秒 + toast 倒计时（3→2→1 秒），3 秒后自动恢复当前时段真实亮度（用 ref 锁住最新值避免 useEffect 时序问题）

五、后端 API 接口

方法路径说明
GET/api/rooms获取所有房间
GET/api/rooms/:id获取单个房间
PUT/api/rooms/:id更新房间（温度/电源）
PUT/api/rooms/:id/mode更新模式（仅总控）
GET/api/settings获取全局设置
PUT/api/settings更新全局设置
PUT/api/settings/mode更新总控模式
GET/api/health健康检查
GET/api/time获取服务器时间

统一响应格式：{ "code": 0, "msg": "SUCCESS", "data": {...} }

温度范围：16–30°C；亮度：0–100；模式枚举：cool / heat / wind / auto；非总控房间调用模式更新返回 403。

六、交互说明

滑动导航

横向 track 视图（3 个）：

当前页面左滑右滑
首页→ 总控面板—
总控面板→ 房间列表→ 首页
房间列表—→ 总控面板

覆盖层（房间详情 / 全局设置）：禁用任何滑动手势，只允许用页面内返回按钮退出。

一键启动/关闭

首页最下方按钮（底部居中），点击后同步所有房间的电源状态 + 总控：

  • 关机时：显示"一键启动"
  • 开机时：显示"一键关闭"（反色填充，图标旋转）
  • 右上角改为全局设置按钮（无外圆边框，与房间列表/主页一致）

总控电源校验

  • 总控（globalSettings.power）关闭时，房间列表或房间详情页单独开启任意房间 → toast "请先打开总控开关"
  • 关闭动作不受总控限制
  • 在 app.tsx 顶层用 useToast 拦截，所有页面都能看到提示

亮度跨时段预览

  • 当前时段滑块（白天调白天/晚上调晚上）→ 直接写入持久配置
  • 跨时段滑块（白天调晚上/晚上调白天）→ 临时预览 3 秒，3 秒后自动恢复当前时段真实亮度
  • 预览期间：屏幕亮度实时跟随滑块 + toast 倒计时
  • 滑块用本地虚拟值显示（不回弹），3 秒后回弹到真实配置位置
  • 跨时段值不会写入持久配置

关机状态

  • 总控/房间面板：面板变暗（opacity 0.5），仅显示时间和电源按钮
  • 首页：不变暗，环境数据正常显示

七、本地部署预览

环境要求：Node.js 18+，npm 或 yarn

前端启动

cd ac-panel/frontend
npm install
npm run dev

访问 http://localhost:5173

注意：不能直接用浏览器打开 index.html，必须通过 npm run dev 启动开发服务器。

后端启动

cd ac-panel/backend
npm install
npm run dev

访问 http://localhost:3001/api/health 验证

生产构建

cd ac-panel/frontend
npm run build      # 输出到 dist/
npm run preview    # 预览构建结果

Vercel 部署

项目通过 Vercel 与 GitHub 仓库关联，每次 git push origin main 自动触发 re-deploy。无需手动操作。
